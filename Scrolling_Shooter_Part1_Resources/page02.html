Variables

Before we will actually create the scrolling shooter game we need to delve a bit into an important concept of GameMaker: Studio, the use of variables. This simple concept will provide a very powerful mechanism to enable much more interesting game play. So what is a variable? It can best be seen as a property of an instance of an object. Let's explain that in a bit more detail...

As you should know, there are certain properties we can indicate when we define an object, for example we can set whether it is visible or whether it is solid. There are also a number of actions that change these, and other, properties. For example there is an action to change the position or the speed of the instance.

Each instance has a number of such properties, and there are also a number of global properties, like the score, that are not related to individual objects, but that are accessible by all of them. All properties are stored in so-called variables that have a name. Here are some of the properties/variables that each instance has:

  • x: the x coordinate of the instance

  • y: the y coordinate of the instance

  • hspeed: horizontal speed (in pixels per step)

  • vspeed: vertical speed (in pixels per step)

  • direction: current direction of motion in degrees, 0 being right, and going anticlockwise to 360 being right again

  • speed: the current speed in this direction

  • visible: whether the object is visible (1, or true) or invisible (0, or false)

  • solid: whether the object is solid (1, or true) or not (0, or false)
And here are some global variables:

  • score: the current value of the score

  • lives: the current number of lives

  • health: the current health of the player (defaults to 100)

  • mouse_x: the x coordinate of the mouse

  • mouse_y: the y coordinate of the mouse

  • room_speed: current speed (FPS) of the room in steps per second

  • room_caption: caption shown in the window title

  • room_width: width of the room in pixels

  • room_height: height of the room in pixels
There are many, many more variables, both of an instance scope, and a global scope (scope is simply a way saying "area of effect"). All of them can be found in the GameMaker documentation which you can open by pressing <F1>. There are actions which manipulate the value of certain variables too, but, as we will see, you can also manipulate them directly. What is better, you can define your own variables and use these in your games as well.

For example, as we will see in the subsequent pages, we want our plane to be able to shoot only once every five steps of the game, so our plane needs a variable to indicate whether it can shoot or not. We use a variable for this property that we will call "can_shoot". "can_shoot" is its variable name and is just a way of identifying information for your game to use.
NOTE: A variable name can only consist of letters and the underscore symbol, and variable names are case-sensitive, so "Can_Shoot" is not the same variable as "can_shoot".
In the Create Event of the plane we will set this variable to true, which is a boolean value (boolean values can only ever be true or false), and when the player wants to fire a shot we check the value of the variable to see whether this is allowed. Whenever a shot is fired we set the variable to false, indicating that shooting is temporarily not allowed.

But how will we permit the player to shoot again? We will use an Alarm Event to set the variable back to true after five steps have passed, and in this way our variable is acting like a switch, where we set it to false on shooting, then your game sets it to true again after a few steps, then the player sets it to false again, etc... The next pages will describe this in more detail, so don't worry if it seems a bit confusing just now, as you'll get more details instructions later!
